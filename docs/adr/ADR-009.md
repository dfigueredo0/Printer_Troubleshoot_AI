# ADR-009: Hybrid memory backend

## Status

Accepted

## Context

Pure in-memory stores were lost between sessions. Pure persistent stores (SQLite) were too slow for hot-path agent queries.

## Decision

Two-tier backend: hot tier is in-memory HNSW for sub-millisecond semantic search; cold tier is SQLite for persistence and large-scale storage. Write-through keeps both in sync.

## Consequences

### Positive

Best of both worlds. 150x faster than SQLite-only search. Data survives restarts.

### Risks and mitigations

Two-tier consistency requires careful write ordering. Mitigated by the unified memory service (ADR-006).
