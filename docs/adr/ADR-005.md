# ADR-005: MCP-first API design

## Status

Accepted

## Context

The CLI was the primary interface, making automation and agent-to-agent calls awkward. Non-Claude tooling couldn't integrate cleanly.

## Decision

All capabilities are exposed as MCP tools first. The CLI becomes a thin wrapper over MCP calls. External systems integrate via the MCP protocol.

## Consequences

### Positive

Native integration with Claude Code and any MCP-compatible client. Tools are self-describing with JSON schemas. Enables the 259-tool surface.

### Risks and mitigations

MCP protocol overhead for simple local operations. Mitigated by local stdio transport.
