# ADR-004: Plugin-based architecture

## Status

Accepted

## Context

Adding new agent types, memory backends, or MCP tools required modifying core files, causing merge conflicts and tight coupling.

## Decision

All extensibility points (agent types, memory backends, MCP tool sets, hook handlers) are defined as plugins loaded at runtime via a registry. Core never imports plugins directly.

## Consequences

### Positive

Third parties can extend the system without forking. Feature flags become trivial. Enables the IPFS plugin marketplace.

### Risks and mitigations

Plugin discovery adds startup latency. Plugin API versioning requires discipline.
