# ADR-003: Single coordination engine

## Status

Accepted

## Context

Multiple competing coordinator implementations (SwarmCoordinator, TaskCoordinator, AgentCoordinator) were creating conflicting state and race conditions.

## Decision

Consolidate to a single `UnifiedCoordinator` as the sole source of truth for swarm topology, task assignment, and agent state. All other coordinators become thin facades.

## Consequences

### Positive

Eliminates split-brain coordination bugs. Simplifies testing. Single place to add consensus logic.

### Risks and mitigations

Single point of failure risk. Mitigated by the consensus and quorum subsystems (ADR-007).
