# ADR-010: Remove Deno support

## Status

Accepted

## Context

Maintaining dual Node.js/Deno compatibility doubled build complexity and blocked use of npm-only packages (better-sqlite3, HNSW bindings).

## Decision

Drop all Deno-specific code paths, compatibility shims, and CI targets. The runtime is Node.js 18+ exclusively.

## Consequences

### Positive

Halves build configuration complexity. Unlocks native npm packages. Eliminates `deno.json` / `deps.ts` maintenance.

### Risks and mitigations

Users on Deno must migrate. Mitigated by providing clear migration notes in MIGRATION_SUMMARY.md.
