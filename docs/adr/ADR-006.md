# ADR-006: Unified memory service

## Status

Accepted

## Context

Six separate memory stores (SQLite, HNSW, file-based, session, learning, swarm) operated independently, causing stale reads and duplicated indexing work.

## Decision

Create a single `UnifiedMemoryService` that routes reads/writes through one interface. HNSW vector index is the primary search backend. Other stores become implementation details behind the service.

## Consequences

### Positive

150x–12,500x search improvement from HNSW. No more cross-store inconsistency. Simpler API for agents.

### Risks and mitigations

HNSW index rebuild time on first run. Mitigated by incremental indexing.
