# ADR-008: Vitest over Jest

## Status

Accepted

## Context

Jest's CommonJS-first model conflicted with the ESM-native codebase, requiring complex transform configs and causing intermittent test failures.

## Decision

Use Vitest as the sole test runner. No Jest dependencies. All test files use `.test.ts` / `.spec.ts` extensions. Mocking via `vi.*` APIs.

## Consequences

### Positive

Native ESM support. 3–10x faster test runs via Vite's module graph. Unified config in `vitest.config.ts`.

### Risks and mitigations

Vitest API differs from Jest in edge cases. Mitigated by a thin compatibility shim for common patterns.
