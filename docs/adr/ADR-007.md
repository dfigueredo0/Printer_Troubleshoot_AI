# ADR-007: Event sourcing for state

## Status

Accepted

## Context

Agent and task state mutations were in-place updates, making it impossible to replay, audit, or roll back swarm decisions.

## Decision

All state changes are appended as immutable domain events to an event log. Current state is derived by replaying the log. Snapshots are taken every 100 events.

## Consequences

### Positive

Full audit trail. Time-travel debugging. Enables the checkpoint/rollback system in `checkpoint-manager.sh`.

### Risks and mitigations

Event log can grow large. Mitigated by snapshotting and the 30-session cleanup in `hook-handler.cjs`.
